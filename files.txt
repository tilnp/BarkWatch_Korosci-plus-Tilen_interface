Data file-i in njihova uporaba


data/heatmap_past_data_no_zeros.csv  (generirano iz heatmap_past_data.csv)
data/heatmap_future_predictions_no_zeros.csv  (generirano iz heatmap_future_predictions.csv)
Stolpci: ggo, odsek_id, leto_mesec, target
Vrstice z target == 0 so odstranjene (hitrejše nalaganje; ničelne vrednosti so implicitno bucket 0)
Generiraj z: python scripts/drop_zero_rows.py target heatmap_past_data.csv heatmap_future_predictions.csv

heatmap_past_data: zgodovinski (izmerjeni) podatki — napaja heatmap za pretekle mesece
heatmap_future_predictions: napovedani podatki — napaja heatmap za prihodnje mesece
  → Najzgodnejši mesec v future datoteki postane FORECAST_START_MONTH (meja na sliderju)
  → Če se mesec pojavi v obeh, OVERLAP_PREFER določi kateri vir zmaga (privzeto: 'predictions')
  → target se deli s povrsino odseka (ha) → relativni posek (m³/ha) za barvanje in prikaz
  → Pragovi za buckete (1–4): auto iz kvantilov ali ročno z HEATMAP_CUSTOM_BREAKS
  → API: /api/heatmap/meta, /api/heatmap?month=..., /api/heatmap/value?odsek=...&month=...&ggo=...


data/odseki_nazivi.csv
Stolpci: ggo_naziv, odsek, povrsina, gge_naziv, ke_naziv, revir_naziv, katgozd_naziv,
         ohranjen_naziv, relief_naziv, lega_naziv, pozar_naziv, intgosp_naziv, krajime, grt1_naziv

Opisni atributi vsakega odseka (ime, površina v ha, GGE, revir, kategorija gozda, ...)
Prikazuje se v details tabeli v levem panelu ko klikneš/poiščeš odsek
Napaja /api/ggo (seznam GGO območij) in /api/odseki/by-key
povrsina se uporabi za izračun relativnega poseka (m³/ha):
  → za barvanje: povprečje čez vse GGO-je z istim odsek_id (POVRSINA_BY_ODSEK)
  → za detail panel: točna vrednost za izbran (odsek, ggo) par — ujema se s prikazanim podatkom
OPOMBA: ne vsebuje vseh odsekov — ni vir za iskanje/suggest (glej odseki_vector_map.mbtiles)


data/odseki_vector_map.mbtiles
Vektorske tile-e za odseke — geometrije poligonov na mapi
Servira se na /tiles/{z}/{x}/{y}
Uporablja se za: risanje odsekov, klikanje, heatmap barvanje (fill-color), highlight izbranega odseka
Ob zagonu server iz teh tiles zgradi bbox cache za zoom na odsek
Primarni vir za autocomplete/suggest iskanje — ODSEKI_BY_GGO se zgradi iz bbox indexa
  → pokriva vse odseke, ne samo tiste v odseki_nazivi.csv
  → odsek ID-ji so normalizirani brez presledkov (npr. "01 58A" → "01058A")
Bbox cache ima verzijsko številko (_BBOX_CACHE_VERSION); ob spremembi se samodejno obnovi


data/odseki_vector_map.bbox_cache.json
Cache bounding boxov za vse odseke (generiran iz .mbtiles ob zagonu)
Pospeši zoom na odsek brez ponovnega dekodiranja tiles
Vsebuje polje _version — ob neujemanju z _BBOX_CACHE_VERSION v serverju se cache samodejno obnovi


data/ggo_vector_map.mbtiles
Meje GGO območij (gozdnogospodarska območja)
Servira se na /ggo-tiles/{z}/{x}/{y}
Prikazan kot zelena (#4ade80) obroba na mapi — layer ggo-outline
Toggle z checkboxom v legendi


data/slovenija_vector_map.mbtiles
Meja Slovenije
Servira se na /slo-tiles/{z}/{x}/{y}
Prikazan kot svetlo modra (#60cdee) obroba — layer slovenija-outline
Toggle z checkboxom v legendi


data/odseki_processed.csv
Vsebuje surove geometrije + vse numerične atribute odsekov (tarife, lzskdv vrednosti, ...)
Ni direktno v uporabi v spletnem serverju — verjetno vhodni file za generiranje .mbtiles in odseki_nazivi.csv (preprocessing)


scripts/drop_zero_rows.py
Pomožni skript — odstrani vrstice kjer je vrednost določenega stolpca enaka 0
Shrani rezultat v novo datoteko z pripono _no_zeros (original ostane nespremenjen)
Uporaba: python drop_zero_rows.py <stolpec> <file.csv> [file2.csv ...]
Primer: python drop_zero_rows.py target heatmap_past_data.csv heatmap_future_predictions.csv


scripts/drop_is_a_prediction.py
Pomožni skript — odstrani stolpec is_a_prediction iz CSV datoteke
Shrani rezultat v novo datoteko z pripono _no_prediction (original ostane nespremenjen)
Uporaba: python drop_is_a_prediction.py <file.csv> [file2.csv ...]
